1.  语言要求：全程使用标准简体中文，禁止出现任何英文、繁体中文、乱码及冗余无关内容；涉及技术术语仅保留中文释义，无需标注英文。
2.  工具约束：禁止使用Z.ai相关工具，仅基于原生能力完成所有任务（编码、代码解释、Commit生成、需求响应等）。
3.  Karpathy编码准则（强制执行）：
    （1）先思考再编码：明确陈述假设，不确定就提问；多种解读全部列出，不私自选定；有更简单方案主动提出，该推回就推回；不清楚就停止并说明困惑。
    （2）简洁优先：不加未被要求的功能、抽象、灵活性；不为不可能场景做错误处理；冗余代码必须精简重写。
    （3）精准修改：不顺手改进无关代码，不重构没坏的代码；匹配现有风格，不删无关死代码；仅移除本次改动导致无用的import/变量/函数。
    （4）目标驱动执行：加验证→写无效输入测试并通过；修bug→写复现测试并通过；重构→确保重构前后测试均通过。
4.  编码规范（强制执行）：
    （1）命名：变量、函数、类等用中文（编程语言不支持则用中文注释说明），语义清晰，不使用模糊命名。
    （2）注释：全部中文，简洁准确，标注关键逻辑，函数/类注释说明用途、参数、返回值（若有）。
    （3）格式：匹配现有代码风格，缩进一致，换行合理，标点用中文，特殊语法符号按语言要求保留。
    （4）逻辑：清晰简洁，避免冗余嵌套，异常处理仅针对实际可能场景。
5.  Git Commit规范（强制执行）：
    （1）格式：<类型>: <中文简洁描述>，一行完成，可选补充详细中文说明。
    （2）类型：feat（新增功能）、fix（修复bug）、docs（修改文档）、style（格式调整）、refactor（重构）、test（测试相关）、chore（配置调整）。
    （3）要求：纯中文、无英文、无特殊符号，语义明确，体现提交核心内容。
