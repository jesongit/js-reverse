# 更新 js-reverse 技能

触发词：`更新 js-reverse`

用途：在**用户明确指定的目标工作目录**基础上，先补齐或更新该目录的 `analysis.md`，再对 `js-reverse` skill、知识库与必要说明做持续优化沉淀。

## 一、硬性前提

用户必须明确提供：

- 目标工作目录

未提供时，必须先询问。

禁止：

- 猜测目标目录
- 默认使用当前目录
- 不读目标目录材料就直接优化 `js-reverse`

## 二、执行目标

每次处理一个具体 JS 逆向工作目录时，都要把结果按下面顺序沉淀：

1. 目标工作目录 `analysis.md`
2. `.claude/skills/js-reverse.md`
3. `.claude/skills/knowledge/`
4. 目标工作目录 `README.md`

其中，**`analysis.md` 是前置步骤，不是可选项。**

## 三、输入要求

用户应提供：

- 目标工作目录（必填）
- 本次希望优化的重点（可选）

若缺少目标工作目录，先问再做。

## 四、强制流程

按下面顺序执行，不允许跳步。

### 第 1 步：读取目标目录与相关材料

至少读取：

- 目标工作目录 `README.md`
- 目标工作目录 `WORKLOG.md`
- 目标工作目录关键源码、验证脚本、结果文件、分析脚本
- 若已存在，读取目标工作目录 `analysis.md`
- `.claude/skills/js-reverse.md`
- `.claude/skills/knowledge/README.md`
- 与本次问题最相关的知识库文件

额外检查规则：

- 若目标目录涉及登录态、动态 env、初始化接口、浏览器上下文恢复，必须检查是否需要更新 `05-project/notion-live-evidence-and-multi-source-env.md`
- 若目标目录涉及 AI 对话网页、SSE 流式接口、OpenAI 兼容封装、模型映射、额度接口、线程详情接口，必须检查是否需要把“最小可用接口集、不同鉴权层、兼容范围边界”提升到 `js-reverse.md` 强制规则

### 第 2 步：先生成或更新目标目录 `analysis.md`

在优化 skill 之前，必须先生成或更新目标工作目录的 `analysis.md`。

要求：

- 优先参考其他工作目录现有 `analysis.md` 的结构与粒度
- 结合 `README.md`、`WORKLOG.md`、关键脚本、结果文件、验证结论整理
- 若已存在 `analysis.md`，基于新增证据补充或修正，不要无脑重写
- `analysis.md` 只写阶段总结与可复用分析，不承载滚动日志

`analysis.md` 至少覆盖：

- 目标
- 最终交付物
- 核心链路 / 接口
- 关键结论
- 边界 / 限制
- 问题复盘
- 方法论

### 第 3 步：归类这次暴露的问题

把问题归类为以下四类：

- **提示词问题**：skill 缺少约束、顺序不清、触发条件不明确
- **知识库问题**：已有方法缺失、专题空白、案例没沉淀
- **结构问题**：skill 冗长重复、章节位置不合理、知识文件组织不清
- **项目问题**：仅对当前任务有价值，不应进入通用知识库

### 第 4 步：决定信息应该落在哪

#### 更新 `js-reverse.md`

仅当满足以下任一条件：

- 同类遗漏重复出现
- 某条规则应该提升为强制约束
- 阶段切换条件不清晰
- 工具使用顺序存在明显更优实践
- README / WORKLOG 记录规则需要加强

#### 更新知识库

仅当满足以下条件之一：

- 具有跨任务复用价值
- 不依赖单一站点业务语义
- 能帮助更快定位、还原或验证
- 能形成独立专题或错题条目

#### 只保留在目标目录 `README.md`

若内容属于以下类型，则不要写入通用知识库：

- 当前站点特有说明
- 临时凭证、常量、路径
- 无法证明可复用的细节
- 只影响当前交付使用方式的稳定说明

### 第 5 步：执行最小必要修改

#### 对 `js-reverse.md`

可做：

- 补规则
- 调顺序
- 去重复
- 合并同类章节
- 强化触发条件

禁止：

- 破坏 7 阶段主脊柱
- 引入与项目无关的大量新规则
- 为单次特例过拟合

#### 对知识库

可做：

- 在已有专题下追加条目
- 新建小型专题文件
- 更新索引文件
- 新增错题本条目

建议长期维护：

- `.claude/skills/knowledge/06-learning/08-mistakes/js-reverse-mistakes.md`

### 第 6 步：按规则判断是否记入错题本

满足以下任一条件时，记录到错题本：

- 同类问题重复出现两次以上
- 问题导致明显误判阶段或误判结论
- 问题不是代码 bug，而是分析路径错误
- 未来再次出现时可以用一条检查项快速定位

记录格式建议：

```md
## Title
- 现象：
- 错误判断：
- 正确做法：
- 快速检查：
- 来源任务：
- 适用范围：
```

## 五、输出要求

执行完后，必须明确说明：

- 更新了哪些文件
- 每个文件为什么更新
- 哪些内容只保留在目标工作目录 README，没有写入通用知识库
- 是否新增或更新了 `analysis.md`
- 是否新增了错题本条目

## 六、硬性约束

1. 禁止把目标站点私有常量、Cookie、密钥写入通用知识库
2. 禁止把单次临时调试过程直接抄进 skill
3. 禁止只改 skill 不说明原因
4. 禁止不读目标工作目录 `README.md` 和 `WORKLOG.md` 就直接优化 skill
5. 如果目标工作目录 `README.md` 缺失，先建议补齐，再继续优化
6. 如果目标工作目录 `WORKLOG.md` 缺失，先创建并记录，再继续优化
7. 文件名和目录名必须使用英文
8. `analysis.md` 未生成或未更新前，禁止开始优化 `js-reverse.md` 与知识库
