# 更新 js-reverse 技能

触发词：`更新 js-reverse`

用途：**必须先读取用户明确指定的工作目录内容**，再对 `js-reverse` skill 进行提示词优化、知识库优化、必要整理与轻量重构，并将值得长期保留的问题与解决经验沉淀到知识库中，形成持续改进闭环。

## 强制前置条件

用户必须明确提供：

- 目标工作目录

未提供时，必须先询问；禁止猜测、禁止默认使用当前目录、禁止跳过读取目录内容直接优化 `js-reverse`。

## 目标

每次在一个具体工作目录完成或推进 JS 逆向任务后，从以下材料中提炼可复用经验：

- 目标工作目录 `README.md`
- 目标工作目录下的请求记录、验证脚本、分析脚本
- 当前任务中暴露出的提示词不足
- 当前知识库覆盖盲区
- 当前 skill 中不清晰、重复、过时的指令

然后把结果分流到：

1. `js-reverse.md` 提示词
2. `.claude/skills/knowledge/` 知识库
3. 目标工作目录 `README.md`（必要时补齐总结）

## 输入要求

用户应提供：

- 目标工作目录（必填）
- 本次希望优化的重点（可选）

若未提供工作目录，必须先询问，不允许猜测。

## 工作流

### 1. 读取材料

至少读取：

- 目标工作目录 `README.md`
- 目标工作目录下关键源码 / 记录文件
- `.claude/skills/js-reverse.md`
- `.claude/skills/knowledge/README.md`
- 与本次问题最相关的知识库文件

### 2. 归类问题

把发现的问题归类为：

- **提示词问题**：skill 缺少约束、顺序不清、触发条件不明确
- **知识库问题**：已有方法缺失、专题空白、案例没沉淀
- **结构问题**：skill 冗长重复、章节位置不合理、知识文件组织不清
- **项目问题**：仅对当前任务有价值，不应写入通用知识库

### 3. 决定落点

#### 应更新 `js-reverse.md` 的情况

- 多次出现同类遗漏
- 某条规则应提升为强制约束
- 阶段切换条件不够清晰
- README 记录规则需要加强
- 工具使用顺序存在明显更优实践

#### 应更新知识库的情况

- 至少具有跨任务复用价值
- 不依赖单一站点业务语义
- 能帮助更快定位、还原或验证
- 可以形成独立专题或错题条目

#### 只留在项目 README 的情况

- 仅当前目标特有
- 只是一组临时凭证、常量、路径
- 无法证明具备复用性

### 4. 形成沉淀

#### A. 提示词优化

可做：
- 补规则
- 调顺序
- 去重复
- 合并同类章节
- 强化触发条件

禁止：
- 破坏 7 阶段主脊柱
- 引入与项目无关的大量新规则
- 为单次特例过拟合

#### B. 知识库优化

可做：
- 在已有专题下追加条目
- 新建小型专题文件
- 新增“错题本”类记录
- 更新索引文件

建议新增一个长期文件用于沉淀高价值问题：
- `.claude/skills/knowledge/06-learning/08-mistakes/js-reverse-mistakes.md`

### 5. 错题本规则

当满足以下任一条件时，记录到错题本：

- 同类问题重复出现两次以上
- 问题导致明显误判阶段或误判结论
- 问题不是代码 bug，而是分析路径错误
- 未来再次出现时可以通过一条检查项快速定位

记录格式建议：

```md
## Title
- 现象：
- 错误判断：
- 正确做法：
- 快速检查：
- 来源任务：
- 适用范围：
```

## 输出要求

执行完后，必须明确说明：

- 更新了哪些文件
- 每个文件为什么更新
- 哪些内容只保留在目标工作目录 README，没有写入通用知识库
- 是否新增了错题本条目

## 约束

1. 禁止把目标站点私有常量、Cookie、密钥写入通用知识库
2. 禁止把单次临时调试过程直接抄进 skill
3. 禁止只改 skill 不说明原因
4. 禁止不读目标工作目录 README 就直接优化 skill
5. 如果目标工作目录 README 缺失，要先建议补齐，再继续优化
6. 文件名和目录名必须使用英文
