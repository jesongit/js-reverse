# 阶段速查

## 整体流程

```
信息登记 → 证据采集 → locate → recover → runtime → validation → 交接
```

## locate（定位）

**入口条件**：目标请求未确认 / 上游依赖链不完整 / 写入边界仍是猜测

**退出条件**：`writer ← builder ← entry ← source` 具体化，下一步阻塞不再是请求发现

**核心问题**：
- 真实目标请求是什么？
- 最终写入边界在哪？
- 上游依赖链是什么？
- 正常态与风控态是否分离？

**工作模型**：
```
writer ← builder ← entry ← source
```

| 汇聚点类型 | 首选观察点 |
|---|---|
| body 字段 | 最终序列化或提交前写入点 |
| header 字段 | header 设置点或最终请求构造 |
| JS 写入的 Cookie | setter 或最终写入点 |
| 响应 Set-Cookie | 网络响应包 |
| WebSocket 帧 | send 前的信封层 |
| worker 回复 | postMessage 契约 |

---

## recover（还原）

**入口条件**：汇聚点/写入边界已确认且真实，下一阻塞是壳层逻辑

**退出条件**：目标字段形成路径已解释充分；或桥接契约和关键算子已足够支持 runtime 或验证

**核心问题**：
- 开哪层？（外层容器 / dispatcher / 状态载体 / 桥接层 / 核心算子）
- 开多深？（A级关键opcode / B级dispatcher+状态载体 / C级最小反编译）
- 何时停？（满足下游需求即可，不追求完整理解）

**优先开启顺序**：
| 症状 | 首先开启的层 |
|---|---|
| 找不到真实可调用路径 | 外层容器 |
| 大型 switch 表或 VM 风格流 | dispatcher 层 |
| 调用可见但参数/状态不透明 | 状态载体 |
| 逻辑仅在 worker/wasm/loader 回调后出现 | 桥接层 |
| 写回点已知但算法不透明 | 核心算子 |

---

## runtime（运行时适配）

**入口条件**：边界和壳层已清晰，但浏览器与本地执行在状态、时序、分支或环境事实上存在分歧

**退出条件**：分歧点明确 + 最小补丁方案可行

**核心问题**：
- 第一个真正的分歧点是什么？
- 保持执行对齐所需的最小依赖集是什么？
- 生命周期产生的事实 vs 可修补的表面事实？

**分歧分类**：
| 类型 | 信号 |
|---|---|
| missing object | undefined window/document/navigator/crypto |
| missing state | 不崩溃但始终失败或走风控态 |
| anti-debugging | 断点冻结、无限 debugger、输出改变 |
| unstable source | 同输入不同输出（时间/随机/种子未冻结） |
| risk branch | 浏览器正常但本地总是走另一条路径 |

---

## validation（验证）

**入口条件**：主要阻塞不再是发现、还原或运行时适配；需要检查点证明

**退出条件**：所有检查点已明确、固定输入已声明、每个确认点有直接证据、剩余缺口已命名

**核心问题**：
- 中间状态是否等价？
- 哪些检查点已确认？
- 剩余缺口是什么？

**验证检查点模板**：
| 检查点 | 浏览器侧 | 本地/恢复侧 | 结果 | 证据 | 剩余缺口 |
|---|---|---|---|---|---|

---

## 阶段交接

每个阶段切换时输出：
```text
--- Stage Handoff ---
From: {上一阶段}
To: {下一阶段}

Proven facts carried forward:
  - 目标请求:
  - 写入边界:
  - 上游依赖链:
  - 样本状态:
  - 已恢复契约:
  - 运行时分歧点:

Open questions for next stage:
  - 问题1
  - 问题2

Invalidated assumptions:
  - 被推翻的假设，或"无"
```

## 复杂度定级

| 级别 | 标签 | 特征 | 预期阶段 |
|---|---|---|---|
| L1 | 透明链路 | 参数明文拼接，无混淆，无环境依赖 | locate → validation |
| L2 | 单层壳 | 简单混淆或 webpack，一层加密，无环境检测 | locate → recover → validation |
| L3 | 多层壳+环境 | JSVMP/wasm/worker + 环境分支 + 反调试 | 完整链路 |
| L4 | 对抗性防护 | 多跳Cookie + 动态代码 + 反调试 + 指纹 + 风控分支 | 完整链路，可能多次迭代 |

---

## 快速决策卡

**当前该进入哪个阶段？**

| 工程状态 | 应进入阶段 |
|---|---|
| 目标请求未从真实样本确认 | evidence（证据采集） |
| 请求已确认但写入边界未知 | locate |
| 写入边界已知但壳层阻碍可调用性 | recover |
| 边界清晰但浏览器与本地执行分歧 | runtime |
| 阻塞是检查点证明而非发现/还原/适配 | validation |

**禁止跳步规则**：
- 写入边界未证明时，不进入 recover
- 还原未完成时，不进入 validation
- 未验证不交付
