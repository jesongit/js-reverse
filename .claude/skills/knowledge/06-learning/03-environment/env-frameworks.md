# 补环境框架评测

> 来源: [多个开源的js补环境框架测试](https://juejin.cn/post/7366082235243741221)

## 概述

开源补环境框架可以大幅减少手动补环境的工作量。本文对比评测 6 个主流框架的功能和使用场景。

## 框架对比总览

| 框架 | GitHub | 适用场景 | 推荐度 |
|------|--------|----------|--------|
| **sdenv** | @wjz8910/sdenv | 直接运行目标 JS | ★★★★★ |
| **qxVm** | @qiegla/qxVm | 日志追踪 | ★★★★ |
| **node-sandbox** | @xiuluotie/node-sandbox | 自定义补环境 | ★★★ |
| **boda_jsEnv** | @smallfawn/boda_jsEnv | 完整环境 | ★★ |
| **catvm** | — | 简单场景 | ★ |
| **v_jstools** | — | 辅助工具 | ★★ |

## 详细评测

### 1. sdenv（推荐）

**特点**：模拟框架，内置浏览器环境，可直接运行目标 JS

**优点**：
- 环境补齐度高，开箱即用
- 支持直接 `require` 目标 JS 文件执行
- 内置常见网站的环境适配
- 更新活跃

**使用方式**：
```javascript
// 安装
// npm install sdenv 或直接使用源码

// 使用 sdenv 运行目标脚本
const sdenv = require('sdenv');
// sdenv 会自动补齐浏览器环境
// 然后直接执行目标 JS
```

**最佳场景**：需要快速运行整段加密 JS，不想手动补环境

---

### 2. qxVm

**特点**：基于 VM2 的沙箱环境，提供详细的日志输出

**优点**：
- 日志系统完善，能看到每个属性的访问记录
- 便于定位环境缺失
- 基于 VM2 隔离性好

**缺点**：
- 部分环境需要手动补齐
- 配置相对复杂

**最佳场景**：需要详细追踪环境访问日志

---

### 3. node-sandbox

**特点**：可自定义的 Node.js 沙箱环境

**优点**：
- 灵活度高，可自定义补环境逻辑
- 提供基础框架，便于扩展

**缺点**：
- 项目已停止维护（abandoned）
- 需要较多手动配置

**最佳场景**：需要高度自定义补环境方案

---

### 4. boda_jsEnv

**特点**：完整的浏览器环境模拟

**优点**：
- 环境补齐较全面

**缺点**：
- 测试时存在运行失败的问题
- 文档不足
- 兼容性有待提升

---

### 5. catvm

**特点**：轻量级补环境方案

**缺点**：
- 环境补齐不完整
- 仅适合非常简单的场景
- 太多功能缺失

---

### 6. v_jstools

**特点**：JS 逆向辅助工具集

**功能**：
- 补环境辅助
- 代码分析
- 与其他框架配合使用

---

## 选择建议

```
                    需要快速运行？
                   /              \
                 是                否
                 |                  |
            使用 sdenv         需要调试追踪？
                                /          \
                              是            否
                              |              |
                          使用 qxVm     需要高度定制？
                                        /          \
                                      是            否
                                      |              |
                                 node-sandbox     手动补环境
```

### 推荐组合方案
1. **新手入门**：sdenv 直接运行，遇到问题用 Proxy 检测
2. **进阶分析**：qxVm 日志追踪 + 手动补齐缺失项
3. **深度定制**：node-sandbox 框架 + 自定义环境模块

## 关键要点总结

- sdenv 是当前最推荐的补环境框架，开箱即用
- qxVm 的日志追踪功能对定位环境缺失非常有用
- node-sandbox 适合需要高度自定义的场景
- 框架不能覆盖所有场景，必要时仍需手动补齐
- 选择框架时应考虑目标网站的检测深度
